# 문제 1. 스마트폰 아이콘/폴더 이동

## 문제 설명

크기 `n` 스마트폰 홈 화면에 `1`번부터 `n`번까지의 아이콘이 있습니다.
초기에는 `i`번 아이콘이 `i`번 위치에 놓여 있습니다.

드래그 규칙은 다음과 같습니다.

1. 아이콘이나 폴더를 드래그해서 **빈 위치**로 옮기면 해당 위치로 이동합니다.
2. 아이콘을 드래그해서 **아이콘이 있는 위치**로 옮기면 해당 위치에 새 폴더가 만들어지고, 두 아이콘이 그 폴더 안에 들어갑니다.
3. 아이콘을 드래그해서 **폴더가 있는 위치**로 옮기면 그 아이콘이 폴더 안에 들어갑니다.
4. 폴더를 드래그해서 **아이콘이나 폴더가 있는 위치**로 옮기면 두 위치의 내용물이 서로 바뀝니다.
5. 아무것도 없는 곳을 드래그하거나, 시작 위치와 도착 위치가 같으면 아무 일도 일어나지 않습니다.

모든 조작이 끝난 뒤, 아이콘 `1..n` 각각이 어느 위치에 있는지 구하려고 합니다.
아이콘 `i`의 최종 위치를 `answer[i-1]`에 담아 반환하세요.

## 제한사항

- `2 ≤ n ≤ 1000`
- `1 ≤ drags.size() ≤ 1000`
- `drags[i] = [start, end]`
- 위치는 `1..n`

## 풀이 개요

위치별 상태를 직접 시뮬레이션합니다.

- 각 위치의 현재 상태를 저장
- 드래그 규칙대로 위치 내용을 갱신
- 마지막에 모든 위치를 순회하며 아이콘이 속한 위치를 복원

## 시간복잡도

- 각 드래그 처리: 위치 상태 업데이트 위주라 충분히 작음
- 전체: `O(n + |drags| + 폴더 내부 원소 총합)`
