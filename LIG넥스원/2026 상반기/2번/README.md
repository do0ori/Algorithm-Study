# 문제 2. 참가할 대회 고르기

## 문제 설명

총 `N`개의 대회가 있습니다. 대회 번호는 `0`번부터 `N-1`번까지입니다.
각 대회는 `M`개의 문제로 이루어져 있으며, 각 문제마다 난도가 주어집니다.

정수 2차원 배열 `contests`가 주어질 때:

- `contests[i][j]`는 `i`번 대회의 `j`번 문제 난도입니다.
- 나의 실력은 `p`입니다.
- 난도가 `p` 이하인 문제만 풀 수 있습니다.

이때 각 대회에서 풀 수 있는 문제 수를 계산한 뒤,
**풀 수 있는 문제 수가 가장 많은 대회 번호 `k`개**를 골라 오름차순으로 return 하려고 합니다.

만약 풀 수 있는 문제 수가 같은 대회가 여러 개라면,
**대회 번호가 작은 대회를 우선 선택**합니다.

---

## 제한사항

- `1 <= N <= 30000`
- `1 <= M <= 10`
- `1 <= k <= min(20000, N)`
- `1 <= p <= 20`
- `1 <= contests[i][j] <= 20`

---

## 입출력 예

| contests                                                   |   k |   p | result         |
| ---------------------------------------------------------- | --: | --: | -------------- |
| `[[1, 5, 2, 3], [9, 9, 9, 9], [1, 9, 3, 9], [4, 3, 2, 1]]` |   2 |   4 | `[0, 3]`       |
| `[[1, 5, 2, 3], [9, 9, 9, 9], [1, 9, 3, 9], [4, 3, 2, 1]]` |   3 |   4 | `[0, 2, 3]`    |
| `[[1, 5, 2, 3], [9, 9, 9, 9], [1, 9, 3, 9], [4, 3, 2, 1]]` |   4 |   4 | `[0, 1, 2, 3]` |
| `[[1, 1, 1], [2, 2, 2], [1, 1, 1]]`                        |   2 |   2 | `[0, 1]`       |

---

## 입출력 예 설명

### 예시 1

실력 `p = 4`일 때 각 대회에서 풀 수 있는 문제 수는 다음과 같습니다.

- 0번 대회: 3문제 (`1, 2, 3`)
- 1번 대회: 0문제
- 2번 대회: 2문제 (`1, 3`)
- 3번 대회: 4문제 (`4, 3, 2, 1`)

가장 많이 풀 수 있는 대회 2개는 3번 대회와 0번 대회이며,
오름차순으로 정렬하면 `[0, 3]`입니다.

### 예시 4

0번, 1번, 2번 대회 모두 각각 3문제를 풀 수 있습니다.
동점일 때는 대회 번호가 작은 순으로 우선 선택하므로 `[0, 1]`이 됩니다.
