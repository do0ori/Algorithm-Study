# 문제 2. 적을 k명 처치하기 위한 최소 초기 체력

## 문제 설명

용사가 여러 명의 적과 싸우려고 합니다.
각 적은 정수 `power`를 하나씩 가지며, 적 하나를 처치하는 규칙은 다음과 같습니다.

현재 용사의 체력을 `H`, 적의 power를 `p`라고 할 때:

- 오직 `H > p` 일 때만 그 적을 처치할 수 있습니다.
- 적을 처치한 후 용사의 체력은 `H + p`가 됩니다.
- 용사의 체력이 **0 이하가 되면 게임 오버**입니다.

즉, 적을 처치한 직후에도 용사의 체력은 항상 **1 이상**이어야 합니다.

적들의 power가 담긴 배열 `powers`가 주어질 때,
정확히 `1명`, `2명`, ..., `n명`의 적을 처치하기 위해 필요한 **최소 초기 체력**을 배열로 구하세요.
여기서 `n`은 `powers`의 길이입니다.

적을 처치하는 **순서는 자유롭게 선택할 수 있습니다.**

## 제한사항

- `1 <= len(powers) <= 200000`
- `1 <= abs(powers[i]) <= 10^9`
- `powers[i]`는 0이 아닙니다.
- 초기 체력은 **1 이상의 정수**입니다.

## 입출력 예

| powers     | result    |
| ---------- | --------- |
| [1]        | [2]       |
| [-1]       | [2]       |
| [2, -1]    | [2, 3]    |
| [3, 4, -2] | [3, 4, 4] |

## 입출력 예 설명

### 예시 1

- power가 `1`인 적을 잡으려면 현재 체력이 `1`보다 커야 하므로 최소 초기 체력은 `2`입니다.

### 예시 2

- power가 `-1`인 적을 잡으려면 `H > -1`은 항상 쉽지만, 처치 후 체력 `H - 1`도 1 이상이어야 합니다.
- 따라서 최소 초기 체력은 `2`입니다.

### 예시 3

- 1명을 잡기 위한 최소 초기 체력은 `2`입니다. (`power = -1`인 적 처치)
- 2명을 모두 잡으려면 예를 들어 `3 -> 5 -> 4`로 가능하므로 최소 초기 체력은 `3`입니다.

### 예시 4

- 1명: `power = -2` 적을 잡으려면 최소 체력 3이 필요합니다.
- 2명: `3, 4`를 순서대로 잡으려면 초기 체력 4가 필요합니다. (`4 -> 7 -> 11`)
- 3명: `power = 3`, `power = 4`, `power = -2` 순으로 잡으면 `4 -> 7 -> 11 -> 9`가 되므로 최소 초기 체력은 4입니다.
